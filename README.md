Репозиторий моего перехода из Middle Frontend Developer в AI Engineer.

Цель — не просто изучить AI, а научиться создавать production-ready AI-системы и подготовиться к собеседованиям на позиции:

- AI Engineer
- LLM Engineer
- AI Backend Engineer
- Generative AI Engineer

---

## Формат обучения

Обучение проходит в формате инженерных спринтов.

Каждый спринт включает:

1. Теорию.
2. Практику.
3. Мини-проект.
4. Code Review.
5. Тест.
6. Мини-собеседование.
7. Ретроспективу.

Следующий спринт начинается только после закрытия предыдущего.

---

## Основные треки

### 1. Core Knowledge

Последовательное изучение необходимых технологий:

- Python
- FastAPI
- SQL
- PostgreSQL
- Docker
- Linux
- Backend Architecture
- LLM
- Embeddings
- RAG
- Vector Databases
- LangChain
- LangGraph
- MCP
- AI Agents
- Deployment
- Monitoring
- MLOps basics

### 2. Capstone Project

Один большой production-oriented AI-проект, который будет развиваться на протяжении всего Bootcamp.

Каждый новый модуль добавляет в capstone новую функциональность.

### 3. Interview Track

Подготовка к собеседованиям идет параллельно с обучением.

Форматы проверки:

- quiz;
- live coding;
- system design;
- code review;
- mock interview.
